# 복잡도

- 알고리즘의 성능을 나타내는 척도
- 시간복잡도와 공간복잡도로 나눌 수 있다.
- 동일한 기능을 수행하는 알고리즘이 있다면 일반적으로 복잡도가 낮을 수록 좋은 알고리즘이다.

## 시간 복잡도

- 코딩 테스트에서는 작성한 프로그램이 모든 입력을 받아 이를 처리하고 실행 결과를 출력하는 데까지 걸리는 시간을 의미한다.
- 시간 복잡도를 표현할 때 빅오표기법을 사용한다.

## 시간 복잡도 표

| 빅오표기법 |      명칭      |
| :--------: | :------------: |
|    O(1)    |   상수 시간    |
|  O(logN)   |   로그 시간    |
|    O(N)    |   선형 시간    |
|  O(NlogN)  | 로그 선형 시간 |
|   O(N^2)   |   이차 시간    |
|   O(N^3)   |   삼차 시간    |
|   O(2^N)   |   지수 시간    |
